## 锁定决策（D-xx）

### D1 产品名与仓库独立
- **决策**：产品名 `htmlshare`。独立目录 `~/Documents/claudeMini/htmlshare`，独立 GitHub 仓库。现有 mdshare 项目**保持原样、继续可用**，两者无共享文件、无共享配置（htmlshare 用 `~/.config/htmlshare/`）。
- **理由**：用户拍板改名；能力已超出 "md"（HTML 直传、多平台发布）；用户明确要求不影响 mdshare 使用。
- **影响范围**：全部代码、配置路径、文档、任务卡。

### D2 代码基线：复制改造 mdshare
- **决策**：以 mdshare 现有模块（convert/enhance/compose/lib/server/test）为基线**复制**进新仓库改造，不是 fork 关联、不是共享依赖。开源协议 MIT。
- **理由**：mdshare 已验证的能力（忠实转换、双模式、直传、门禁、版本留底）占新产品 P0 的大半，复制起步比重写快一个量级；复制而非引用保证两项目互不影响（D1）。
- **影响范围**：M0/M1 全部任务卡的「步骤」均为移植改造而非从零实现。

### D3 分层架构与适配器接口
- **决策**：五层：`core`（转换/组装/加密）→ `adapters`（发布目标适配器）→ `skill`（各 agent 封装）→ `server`（开源自托管服务端）→ `cloud`（闭源增值服务）。所有发布目标实现统一适配器接口（见 docs/04 §5），CLI 只面向接口编程。
- **理由**：适配器隔离是「增值服务弱耦合」「多平台支持」「平台 API 变更风险隔离」三个需求的同一个解。
- **影响范围**：docs/03 架构、docs/04 契约、P 系列全部任务卡。

### D4 保密边界双轨制
- **决策**：分享默认带访问码，按目标分两轨：
  - **服务端目标**（selfhost/cloud）：4 位数字码 + 服务端限速防爆破（延续 mdshare 体验）。
  - **静态目标**（Vercel/Cloudflare，无服务端门禁）：页面本体客户端加密（AES-256-GCM，PBKDF2-SHA256 ≥600k 迭代），访问码默认 8 位 Crockford Base32（约 40 bit 熵）。
  - 两轨统一用户体验：「链接 + 访问码」。`--code <自定义>` 可指定，`--public` 关闭门禁。
- **理由**：静态托管无法做服务端限速，4 位码可被离线秒破，必须加长；40 bit + 慢 KDF 使离线爆破成本达年级别，匹配「轻量保密、非强机密」定位（诚实声明写入文档与页面）。
- **影响范围**：C-08 加密器、P-02/03、S-02、docs/02 门禁页、docs/10 风险表。

### D5 模板 × 风格正交体系
- **决策**：**模板 = 信息架构**（内容怎么组织），**风格 = 视觉主题**（长什么样），两者正交可自由组合。首发 5 模板：通用文档、会议纪要、方案提案、教程指南、发布公告；4 风格：临床卡片、极简黑白、杂志编辑、暗色科技。默认由宿主 agent 按内容自选（规则见 docs/05），`--template` / `--style` 可指定，config 可设默认。
- **理由**：用户已定决策（多模板多风格、AI 自选、可指定）；正交设计使 5×4=20 种组合只需 5+4=9 份实现。
- **影响范围**：C-05/06/10~14、docs/05 选择表、SKILL.md。

### D6 增强由宿主 agent 完成，零 LLM API 配置
- **决策**：LLM 增强（重点提炼、TL;DR、重排、模板填充）由**运行 skill 的宿主 agent 本身**按 SKILL.md 指令完成，产出符合 docs/05 输出 Schema 的增强片段；CLI 只做确定性工作（忠实转换、组装、加密、上传）并校验 agent 产出。**不需要用户配置任何 LLM API Key**。agent 未产出或产出校验失败 → 自动降级为仅忠实版，发布照常成功。
- **理由**：这是「尽可能少设置、一键使用」的关键——mdshare 老路（配 LiteLLM）是首次使用最大摩擦点；frontend-slides/baoyu 系热门 skill 全部采用宿主 agent 生成内容的模式；宿主 agent（Claude/Codex 等）本就是用户已付费的最强模型。
- **影响范围**：docs/05 全文、C-04 组装器（含校验）、K 系列 SKILL.md、失败兜底表。

### D7 发布目标自动探测
- **决策**：未指定 `--target` 时按序探测：① config 中已记住的默认目标 → ② selfhost 已配置 → ③ cloud 已登录 → ④ `vercel whoami` 成功 → ⑤ `wrangler whoami` 成功 → ⑥ 全无则输出一次性引导。引导文案优先说明：有自己的 VPS/主机则配置 `selfhost`；没有或想跳过主机配置则走 Cloudflare Pages；Vercel 与官方云仍作为可选目标。用户选定后写入 config 记住。
- **理由**：把「配置」变成「探测 + 记住」，符合零摩擦目标；已有凭据的用户第一次就能直接发。
- **影响范围**：P-04、docs/02 首次使用流程。

### D8 稳定链接策略
- **决策**：同一源文件重复发布，链接不变、只更新内容。服务端目标：id 固定、旧版留底（延续 mdshare）。静态目标：固定 project（如 `htmlshare-pages`）+ 每份分享一个子路径 `/s/<id>/`，本地维护站点镜像目录，每次发布全量重部署该 project。映射关系记录于本地 manifest（源文件绝对路径 → 目标+id）。
- **理由**：「分享链接不变、永远打开最新版」是 mdshare 已验证的核心体验，必须在静态平台上等价复刻。
- **影响范围**：P-02/03、lib/manifest、docs/04 §2。

### D9 云协议 = 自托管协议超集（弱耦合的实现方式）
- **决策**：增值云服务的上传/门禁/版本 API 与开源自托管服务端**完全同构**（docs/04 §6），云端仅叠加：账号认证、配额、计划限制、控制台。对 CLI 而言 cloud 只是「一个带登录的 selfhost 端点」。云服务代码放独立私有仓库 `htmlshare-cloud`（基于开源 server 扩展），开源仓库只含协议文档。
- **理由**：用户红线「增值服务有问题或不上线，不影响其他部分」——协议同构 + 适配器隔离 + 仓库分离，三重保证；同时开源用户随时可迁出到自托管，建立信任。
- **影响范围**：docs/04 §6、V 系列全部、P-05。

### D10 商业模式：免费额度 + 订阅 Pro
- **决策**：官方 Studio/云服务免费层（额度见 Q3）+ 付费扩容。卖点：零运维、免 VPS、稳定域名、服务端门禁（4 位码体验）、版本历史、控制台管理。免费额度不足时，用户可以付费扩容，或切换到自己的 Vercel / Cloudflare / 兼容服务端账号。
- **理由**：用户已定方向（增值 HTML 部署服务）；免费层是获客漏斗也是零配置候选默认目标（Q5）。
- **影响范围**：V-03/04/07、docs/10 成本测算。

### D11 多 agent 安装与分发
- **决策**：Claude Code 的 `SKILL.md` 为**主源格式**；`install.sh`（curl 一行式）自动探测本机已装 agent（`~/.claude/skills/`、`~/.codex/`、OpenClaw/Hermes 约定目录），把 skill 装到所有检测到的位置，各家差异由安装器生成对应封装（Codex 用 AGENTS.md 片段注入等），核心逻辑始终走同一个 CLI。
- **理由**：一份核心 + 薄封装，避免四份维护；curl 一行安装是热门 skill 的事实标准。
- **影响范围**：K 系列全部。

### D12 输出永远是单文件自包含 HTML
- **决策**：所有产出（忠实版、增强版、加密壳、门禁页）均为零外部请求的单文件 HTML：CSS/JS 内联、图片转 data URI（超限则告警）。HTML 直传输入同样内联其本地相对 `<img>`（复用 Markdown 路径逻辑：2MB 上限，缺失/超限/不支持则告警并保留链接）；不含本地图片的 HTML 原样直传、逐字节不变，仅在需要加密时包壳。
- **补充（2026-07-09）**：早期方案曾对直传 HTML 完全「不改字节」，导致带本地图片的页面上传后图片全断。改为直传路径也内联本地图片，与「单文件自包含」的核心原则一致；byte-exact 保证仅对本身已自包含（无本地相对图片）的 HTML 成立。
- **理由**：单文件是「随处可托管、打开即快、离线可存」的根基，也是 mdshare 已验证约定。
- **影响范围**：C 系列全部、非功能指标。

### D13 存储选型
- **决策**：开源自托管服务端延续文件系统存储（`data/<id>/v<n>.html` + `meta.json`），零数据库依赖；云服务用 SQLite（better-sqlite3）管账号/配额/页面索引，页面内容仍存文件系统。
- **理由**：自托管零依赖=部署零摩擦；云端需要事务和查询但单机 SQLite 在预期规模（万级用户）内绰绰有余（YAGNI）。
- **影响范围**：S-01、V-01、docs/04 §7。

### D14 技术栈约束
- **决策**：Node.js ≥ 20，CLI 与服务端均零框架（node:http、node:test）；运行依赖仅限 markdown-it 系、sanitize-html、better-sqlite3（仅 cloud）；前端零依赖全内联。禁止引入构建工具链（无 bundler/无 TS 编译步骤）。
- **理由**：延续 mdshare 极简基因；skill 分发场景下 `npm install` 越轻安装摩擦越小；执行方为 AI agent，栈越简单出错面越小。
- **影响范围**：全部任务卡的实现约束。

### D15 界面原型先行门禁（2026-07-04 追加）
- **决策**：新增 docs/02A 界面设计系统（tokens 具体值 + 组件规范 + 覆盖清单 + 防 AI 味负面清单）与 U-01~U-03 原型卡：静态高保真原型（02A §5 全覆盖）经用户确认（U-03 门禁）后，界面实现卡（C-05/C-06/C-10~C-14）方可开工；原型是实现的黄金标杆。非界面卡不受门禁约束。
- **理由**：plan-product skill 升级（执行期界面质量问题的教训）：审美决策必须在原型期与用户锁定，不留给执行期模型。
- **影响范围**：docs/02A、docs/06 U 系列、TASKBOARD 依赖更新、AGENTS.md 原型门禁协议、M1 出口标准。

### D16 开源交付边界与官方云默认项（2026-07-05 追加）
- **决策**：正式开源版本的核心交付是跨 agent skill、CLI、设置/配置能力、模板/风格渲染、静态平台适配器与服务端协议适配器；**不把官方云服务端代码作为开源产品的一部分发布**。用户若选择自己的服务器，需要在设置页/配置项中填入兼容服务端的地址和凭据后发布；用户若选择官方 Studio/云服务，则开源版提供一个默认选项，走 `cloud` 登录/发布流程，用户无需自行部署或配置服务器，默认获得 100MB 免费空间。
- **理由**：保持开源产品轻量可信，同时把“零配置发布”落到官方托管服务；官方服务端由项目方运营和迭代，避免把商业服务实现与开源客户端强耦合。
- **影响范围**：D3/D9/D13 的发布边界解释、P-04/P-05、C-15 设置、G-01 README、G-04 发布包、V 系列云服务；后续 release 前需检查开源包是否泄漏或误宣称包含官方服务端实现。

### D17 访问过期＝到期即删（2026-07-10 追加）
- **决策**：分享支持过期时间 `expiresAt`。**过期即删**而非「挡在过期墙后」——
  - **服务端目标（selfhost/cloud）**：访问到期页 → 服务端软删并返回 410；每日清扫任务 `expireDue` 也主动软删到期页（无人访问也回收）。软删走既有 7 天宽限可恢复，宽限期从**到期时刻**起算（`deletedAt = expiresAt`）。
  - **静态目标（vercel/cloudflare）**：无服务器强制。双保险——真删靠 `htmlshare sweep`（扫本地 manifest，`unpublish` 已过期项）；托底靠发布时注入的页内到期守卫 JS（到点隐藏内容显示「已过期」）。**守卫可被查看源码/关 JS 绕过，是防君子的托底，非安全边界**；如设了访问码则页内为密文，绕过也只得密文。
- **发布时确认**：`--expires 7d|24h|2026-08-01` 或 `--no-expires`；均未给且有 TTY 则交互询问，非 TTY（脚本）默认永不过期以免挂起；skill 侧要求 agent 发布前先问用户。始终回显 `EXPIRES:` 确认行。事后改期用 `htmlshare expire <id> <期限|--off>`（PATCH /meta，仅服务端目标）。
- **理由**：与 D12「单文件自包含」及既有软删/清扫机制天然契合，复用而非新造；对静态的诚实降级与 D4 加密壳同构。
- **影响范围**：server（store/server.js/pages.js）、CLI（publish/expire/sweep/list）、适配器 publish+setExpiry、manifest、docs/04 契约、SKILL.md；cloud 私库需同步 `expiresAt` 字段与端点（后续单独一批）。

### Q1 云服务域名与品牌
- **推荐方案**：正式域名建议 `htmlshare.app`（或 .cc/.io，以可注册为准）；上线前过渡期用现有服务器 IP+端口联调。

- **可逆性**：高（BASE_URL 为配置项，换域名只改配置+DNS）。

### Q2 支付渠道
- **推荐方案**：已确认采用「微信收款 + 手动开通」作为首发过渡方案；量起来后再接正规渠道（如 Stripe 或微信/支付宝商户能力）。
- **最晚确认点**：V-07 开始前。V-07 之前免费层即可完整运营。
- **可逆性**：中（计费逻辑通过 plan 字段与限制引擎隔离，换渠道不动核心，但用户迁移有成本）。

### Q3 免费/付费额度数值
- **推荐方案**：已确认官方 Studio/云服务默认提供 **100MB 免费空间**；CLI/设置页需要明确提示“当前有 100MB 免费空间，更多空间需付费扩容，或设置自己的 Vercel / Cloudflare 账号”。首发付费形态为手动开通扩容（见 Q2），具体价格和扩容档位可在 V-04/V-07 中配置化保留。
- **最晚确认点**：V-04（计划限制执行）开始前。
- **可逆性**：高（全部为服务端配置值，随时调整，只能放宽不宜收紧已有用户）。

### Q4 GitHub 归属与仓库名
- **推荐方案**：已确认先在用户个人 GitHub 账号 `neowangx` 下建立私有仓库 `htmlshare`，用于当前阶段 git 管理与远端备份；后续公开发布前再按 G-04 决策切换可见性或迁移。
- **最晚确认点**：G-02（建仓推送）前。
- **可逆性**：高（GitHub 支持仓库改名/迁移且自动跳转）。

### Q6 设计方向确认（U-01 比选）
- **推荐方案**：按 docs/02A §1「内容优先的编辑排版」方向与 §2 clinical 缺省 tokens 先行制作样张。
- **最晚确认点**：U-03 门禁通过前均可免费更换方向（更换只重做样张与原型，不涉及任何实现代码）。
- **可逆性**：高（门禁前）→ 低（门禁后，C 系列开工即锁定）。

### Q5 云免费层是否成为零配置默认目标
- **推荐方案**：是——官方 Studio/云服务上线后，未配置任何目标的新用户默认引导注册云免费层（30 秒设备码登录），默认获得 100MB 免费空间，使「装完即发」真正零配置；若需要更多空间，提示付费扩容或设置自己的 Vercel / Cloudflare。
- **最晚确认点**：M5 完成后的 SKILL.md 迭代（不阻塞任何 M0~M5 任务卡）。
- **可逆性**：高（只改 P-04 探测顺序与引导文案）。
