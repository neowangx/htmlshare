> 状态：**已实现并合入（2026-07-11，D18）**。这是对「增强版渲染」的一次大改，**不做向后兼容**（现无实际用户）。落地代码见 `src/a2ui/`（render/catalog/chart）、`src/compose.js`；契约见 docs/04 §4/§8。
> 前置阅读：docs/00（D5/D6/D12/D14）、docs/04 §4 增强片段契约、docs/05 AI 增强设计。

## 1. 背景与目标

- 要转 HTML 的内容形态千差万别，现状「整篇选 1/5 模板 × 1/4 风格」粒度太粗，模型无法按内容细粒度调度排版。
- 高德 **AGenUI** 把这件事做成了标准：底座是 Google **A2UI v0.9** 协议——「模型如何用 JSON 描述界面」；渲染层是把抽象组件翻译成目标平台组件的「翻译器」。AGenUI 写的是**原生**渲染器。
- **本方案 = 采用 A2UI 协议 + 自己实现一个「静态 HTML/CSS 渲染器」**。不引入任何 GenUI SDK/运行时。
- 收益：(a) 模型按内容拼组件树，排版表现力大增；(b) 对齐正在成为事实标准的 A2UI，同一份模型输出未来可复用到原生渲染；(c) 比现状更安全——模型只给**类型化组件+结构化属性**，HTML 全由我们产出。

## 3. 什么是「A2UI 静态子集」

A2UI v0.9 原生是**流式、可数据绑定、扁平列表 + ID 引用建树**的协议（消息类型 createSurface / updateComponents / updateDataModel；`Dynamic*` 属性可取字面量 / JSON Pointer 路径 / FunctionCall）。htmlshare 是一次性静态渲染，取其**收敛快照**：

1. **只取最终态**：一份 `{ root, components[], dataModel }` 快照，忽略流式消息封装。
2. **发布时解析数据绑定**：把所有 `Dynamic*`（字面量/路径）对着 `dataModel` 解析成定值——客户端不需要 data model。
3. **丢弃交互**：`FunctionCall`/action/事件一律忽略或降级（静态页无行为）。
4. **建树**：按 ID 引用（邻接表）从 `root` 建组件树，逐组件走 §4 目录渲染成静态 HTML，套主题 CSS 变量。

htmlshare 消费的形状（对齐 A2UI，字段名随 v0.9 组件库 a2ui.org/reference/components 收敛）：

```jsonc
{
  "protocol": "a2ui/0.9-static",     // 我们消费的静态 profile 标记
  "theme": "editorial",              // 主题层：clinical|minimal|editorial|darktech|auto
  "title": "产品评审会纪要",
  "root": "c0",
  "dataModel": { "rate": "92%" },
  "components": [
    { "id": "c0", "component": "Column", "children": ["c1","c2","c3","c4"] },
    { "id": "c1", "component": "Hero", "kicker": "产品评审", "headline": "2026 Q3 评审结论", "meta": "2026-07-11" },
    { "id": "c2", "component": "StatGrid", "items": [ { "value": {"$path":"/rate"}, "label": "完成率" } ] },
    { "id": "c3", "component": "Callout", "tone": "warning", "richText": "预算需在<b>周五</b>前确认" },
    { "id": "c4", "component": "Chart", "kind": "bar", "series": [ { "label":"A","value":12 }, { "label":"B","value":8 } ] }
  ]
}
```

- `{"$path":"/rate"}` = A2UI Dynamic 的 JSON Pointer，发布时解析为 `"92%"`。
- 富文本字段（`richText`）仍走 `sanitizeEnhanced` 白名单。
- 与现状一样：坏 JSON / 校验失败 → 降级回忠实版（D6）。可借鉴 AGenUI 的「智能 JSON 修复」加一层宽容补全。

## 4. 组件目录与静态渲染映射（首批）

| A2UI 组件 | 静态渲染 | 备注 |
|---|---|---|
| `Text`（variant h1/h2/body/caption） | 语义标签 + 字阶 | |
| `RichText` | 净化后的 HTML 片段 | 复用 sanitizeEnhanced |
| `Column` / `Row` | flex 纵/横排容器 | |
| `List` | `<ul>`/有序列表 | 支持数据绑定模板 |
| `Card` / `Divider` | 面板 / `<hr>` | |
| `Image` | `<img>`（本地图片沿用内联 data URI；远程 https 允许） | 复用 convert 的图片内联 |
| `Table` | 表格 CSS | |
| `Hero`（扩展） | 标题区（kicker/headline/meta） | htmlshare 扩展块 |
| `StatGrid`（扩展） | 指标网格卡片 | |
| `Callout`（扩展，tone info/warning/success/danger） | 带色左边框提示块 | |
| `Quote`（扩展） | 引述块 | |
| `Timeline`（扩展） | 时间线 | |
| `Tabs` | **CSS-only**（radio+label，零 JS） | 无 JS 也能切换 |
| `Chart`（kind line/bar/pie） | **发布时生成内联 `<svg>`** | 见 §4.1 |
| `Button` | 有 URL → `<a>` 按钮样式；纯 action → 降级为文本/丢弃 | |
| `AudioPlayer`/`VideoPlayer`/`Lottie` | 降级：链接或静态封面图 + 说明 | 纯静态零 JS 决策 |

扩展块（Hero/StatGrid/Callout/Quote/Timeline）走 A2UI 的自定义/扩展组件命名空间（AGenUI 也是这么扩到 20+ 的）。

### 4.1 Chart → 内联 SVG

- 新模块 `src/a2ui/chart.js`：输入 `{kind, series/points, axis?}`，**发布时**产出确定性内联 `<svg>`（折线/柱/饼），零脚本、零外链、可打印。
- 坐标/配色取当前主题 CSS 变量（`--hs-accent` 等）。上限：数据点/系列数设阈值，超限降级为表格 + 告警（沿用 IMAGE 超限的诚实降级风格）。

## 5. 不变量与安全（必须守住）

- **D12/D14**：单文件、零外部请求、零运行时框架、无构建工具。交互/媒体一律 §4 降级，绝不引入框架或外链。
- **Altitude / 白名单**：模型只产出**类型化组件 + 结构化属性**；HTML 由我们的渲染器生成。唯一的自由文本入口是 `RichText/*.richText`，强制走 `sanitizeEnhanced`。相比现状「模型写 section HTML」更安全。
- **可访问性红线**（全局规则）：语义标签、对比度、`prefers-reduced-motion`、打印展开——沿用 compose 现有约定。

## 6. 架构与文件改动

```
模型 → A2UI 静态子集 JSON
          │  parse + 校验 + 数据绑定解析（发布时）
          ▼
   src/a2ui/render.js      新：邻接表建树 + 逐组件翻译成静态 HTML
   src/a2ui/catalog/*.js   新：每个组件的 HTML/CSS 生成器（含扩展块）
   src/a2ui/chart.js       新：Chart → 内联 SVG
          │
          ▼
   src/compose.js          改：增强块改调 a2ui/render；忠实版/包壳/主题不变
   src/styles/*            改：clinical/minimal/editorial/darktech 变为主题层（CSS 变量），套在组件之上
```

- **新增**：`src/a2ui/`（render + catalog + chart）。
- **删除**（无向后兼容）：`src/templates/`（registry + 5 模板 + slot）、`enhanced.json` v1 的 shape/校验、compose 里的 template/remap 分支。
- **改**：`src/compose.js`（增强分支换成 a2ui 渲染）、`src/styles/*`（收敛为纯主题变量）、`src/lib/sanitize.js`（RichText 复用，规则基本不变）。
- **文档/指引**：`docs/04` 增强契约整节改写为 A2UI 静态子集；`docs/05` 改为「如何让模型产出 A2UI」；`SKILL.md` 教模型输出组件树并给样例；`docs/00` 记 **D18**（A2UI 静态渲染）。
- **云私库 `htmlshare-cloud`**：内容无关（只存/发 HTML 字节），基本不受影响；仅文档口径同步。

## 8. 风险与开放问题

1. **A2UI 仍在演进**（v0.9）：字段可能变。对策：只锁定「静态子集」的稳定部分，渲染器内做一层薄适配，把协议细节隔离在 `a2ui/` 里。
2. **模型能否稳定产出合法 A2UI**：靠 SKILL.md 强样例 + 校验 + 智能修复 + 降级回忠实兜底（永不发布失败，D6）。
3. **中文排版**：字阶/行高/标点沿用现有风格资产（PingFang/Noto）。
4. **降级边界**：交互/媒体降级规则需逐组件写清并在发布时给 stderr 提示（沿用 IMAGE/EXPIRES 的诚实告知风格）。
5. **迁移成本**：删除 5 模板会牵动一批测试与 docs；因无用户，采硬切换、集中重写。
