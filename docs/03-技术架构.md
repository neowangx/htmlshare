# 03 技术架构

> 必读前置：docs/00（D2/D3/D6/D9/D12/D13/D14）、docs/04（契约细节以 04 为准）

## 1. 架构总览（文字版）

```
┌─ 宿主 Agent（Claude Code / Codex / OpenClaw / Hermes）
│   读 SKILL.md → 生成增强片段 enhanced.json（可选，D6）
│   └─ 调用 ▼
┌─ htmlshare CLI（开源，Node ≥20，零框架 D14）
│   bin/htmlshare.js  子命令: publish / list / unpublish / config
│   ├─ core
│   │   convert.js   md → 忠实 HTML 片段（markdown-it + sanitize 白名单）
│   │   compose.js   忠实版 + 增强片段 + 模板 + 风格 → 单文件 HTML（含 toggle）
│   │   encrypt.js   静态目标加密壳（AES-256-GCM，D4）
│   │   templates/   5 模板 = HTML 结构骨架 + 组件槽位（D5）
│   │   styles/      4 风格 = CSS 变量主题（D5）
│   ├─ adapters（统一接口，见 docs/04 §5）
│   │   selfhost.js  → 开源服务端 API
│   │   cloud.js     → 增值云（= selfhost 协议 + 设备码登录，D9）
│   │   vercel.js    → 本地站点镜像 + vercel CLI 部署（D8）
│   │   cloudflare.js→ 同上，wrangler pages
│   └─ lib
│       config.js    ~/.config/htmlshare/config.json
│       manifest.js  ~/.config/htmlshare/manifest.json（源文件→目标+id+码）
│       sitedir.js   ~/.local/share/htmlshare/sites/<target>/ 静态站点镜像
│
├─ 开源自托管服务端 server/（可选组件）
│   node:http 单文件级实现；文件系统存储（D13）；Docker 部署
│   职责: 上传鉴权(token) / 门禁(4位码+限速) / 版本留底 / 撤回
│
└─ 增值云 htmlshare-cloud（私有仓库，D9 弱耦合）
    = server/ 同构 API + 账号(设备码) + 配额 + 计划 + 控制台 + SQLite 索引
```

**弱耦合保证（D9）**：CLI 对 cloud 的全部认知 = 「一个实现了 docs/04 §6 协议、多一步登录的端点」。cloud 仓库删除、服务下线，开源侧不改一行仍完整可用。

## 2. 选型与理由

| 选择 | 理由（均受 D14 约束） |
|------|---------------------|
| Node ≥20 + node:test | agent 生态默认有 node；零测试框架依赖 |
| markdown-it 系 + sanitize-html | mdshare 已验证（D2）；sanitize 是安全红线不自造 |
| 零 Web 框架（node:http） | 服务端就 6 个路由，框架是负资产 |
| 静态目标走官方 CLI（vercel/wrangler）而非 REST API | 官方 CLI 自带登录态与项目管理，免我们管 token；API 变更风险转嫁给官方 CLI |
| WebCrypto（浏览器端解密） | 零依赖内联，所有现代浏览器可用 |
| better-sqlite3 仅用于 cloud | 开源侧零 native 依赖，安装无编译摩擦 |

## 3. 发布流程状态机

```
IDLE
 └─ publish 触发
COLLECT     读输入(md|html)、config、manifest；html 输入 → 跳到 COMPOSE(直传模式)
CONVERT     md → 忠实 HTML 片段            失败 → FAIL(输入错误,原文报错)
ENHANCE?    存在 enhanced.json(agent 产出) → 校验(docs/05 §5)
             ├─ 通过 → 带增强片段
             ├─ 失败 → 重试1次 → 仍失败 → 忠实版继续(E1)
             └─ 不存在 → 忠实版继续(E8)
COMPOSE     模板+风格+双模式外壳 → 单文件 HTML → 体检(体积/外链) 超限告警(E6)
ENCRYPT?    目标为静态 且 非 --public → 加密壳(D4)
RESOLVE     选目标(D7: 记住的→selfhost→cloud→vercel→wrangler→引导退出)
UPLOAD      调适配器 publish()             失败 → 暂存产物+重试2次 → FAIL(E2/E3, 产物保留)
RECORD      写 manifest(源路径→目标,id,码) → 输出链接+访问码 → DONE
```
任何 FAIL 均保留已完成的中间产物于临时目录，重跑跳过已完成阶段（E2 文案承诺的依据）。

## 4. 部署方案

- **CLI/skill**：无部署。`install.sh` 复制文件 + `npm install --omit=dev`（D11）。
- **自托管服务端**：`server/deploy.sh <user@host> <port> <token>`，Docker 优先，数据卷挂 `data/`。升级=换镜像，数据不动。
- **增值云**：Docker 部署到自有服务器（**执行部署前必须读 `~/.claude/infrastructure.md` 取凭据与现状，部署后回写**——执行 agent 注意）。域名待 Q1，之前用 `BASE_URL` 配置占位。反代 TLS 用 Caddy（自动证书，比 nginx 配置少）。

## 7. 安全边界（红线摘要，详见 docs/10）

- 所有 md 转换产物过 sanitize-html 白名单；HTML 直传不净化但**永不与服务端登录态同源共享 cookie**（门禁 cookie 设 `Path=/s/<id>`、`SameSite=Strict`）。
- 上传一律 Bearer token；服务端解锁限速；静态目标加密参数按 D4 不得下调。
- 分享页默认 `<meta name="robots" content="noindex">`。
