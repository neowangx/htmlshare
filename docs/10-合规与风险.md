# 10 合规与风险

> 必读前置：docs/00（D4/D9/D10）。用户判定本产品不涉及重合规领域（无儿童/医疗/金融），本文覆盖仍然真实存在的横切风险。

## 1. 内容安全（云服务是内容托管方，躲不开）

- **风险**：用户经云服务托管违法/侵权内容，责任落在运营者。
- **措施**（MVP 级，够用为度）：
  - 服务条款页（注册即同意）：禁止违法内容，运营方有权删除与封禁——文本进 NEEDS_HUMAN（法务文本人类审一遍）。
  - 每个云托管页面页脚含举报入口（mailto，含页面 id）；控制台后台 `tools/takedown.js <id>`（下架 + 封 token）。
  - 全部分享页 `noindex`（docs/04 §10）——不进搜索引擎，大幅降低被动传播风险。
  - 开源自托管/Vercel/CF 目标：内容责任在用户自己与平台条款之间，htmlshare 仅为工具（README 声明）。

## 2. 平台条款与依赖风险

| 依赖 | 风险 | 应对 |
|------|------|------|
| Vercel/CF 免费额度 | 条款变化、免费额度收紧、CLI 部署被限 | 走官方 CLI（条款内正当用法）；适配器隔离（D3），任一平台失效只损失一个目标；云服务是自有兜底 |
| vercel/wrangler CLI 输出格式变化 | URL 解析失败 | P-02/03 解析失败时降级打印原始输出并提示手动取 URL；CI 定期跑联调冒烟 |
| 境内服务器 + 域名 | 网站备案要求 | Q1 选域名时一并决策：备案 或 换境外 VPS 或 Cloudflare 代理——进 NEEDS_HUMAN 与 Q1 绑定 |

## 3. 成本测算

**用户侧单次发布成本**：agent token 消耗（用户自己的订阅，增强一篇 3000 字文档约 1~2 万 token）+ 免费托管额度。→ 对用户近似零边际成本，这是产品成立的前提。

**云服务运营成本**（推荐额度 Q3 下）：
- 存储：免费用户上限 20 页 × 5MB = 100MB/人；1000 免费用户 ≈ 100GB 上限、实际按 10% 填充率 ≈ 10GB——现有服务器磁盘可承载。
- 带宽：单页 ≤5MB、noindex 无爬虫流量，预计 1000 用户月带宽 <100GB。
- 固定成本：服务器（已有）+ 域名（~¥100/年）+ SMTP（Resend 免费层 3000 封/月够用）。
- **结论**：M5 阶段月增量成本 ≈ 域名摊销，盈亏平衡点极低；风险在爆量带宽，应对=免费层单页体积硬限 + 访问计数告警阈值（V-03）。
