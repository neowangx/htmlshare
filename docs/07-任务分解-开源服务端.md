# 07 任务分解 — 开源自托管服务端（S）

> 工作目录：开源仓 `server/`。基线：mdshare `server/`（只读参考，严禁修改原目录）。
> 通用约束：D13（文件存储）、D14（node:http 零框架）；契约以 docs/04 §6.1/§7.1 为准。

## 依赖图

```
S-01 ── S-02 ── S-03 ── S-05
                 └───── S-04
```
S-04 与 S-05 可并行。

### S-01 服务端移植与改名
- 里程碑 M1 ｜ 依赖 C-01 ｜ 必读：docs/04 §6.1/§7.1、mdshare server/ 全部
- **目标**：`server/server.js` 在新仓库可跑：上传（token）、门禁、版本留底、基础路由。
- **步骤**：复制 mdshare server（含 test）；全部标识改 htmlshare；存储布局改为 docs/04 §7.1（meta.json + v<n>.html，codeHash 用 scrypt）；.env 按 docs/03 §6。
- **验收**：移植测试全绿；`PORT=0 node server/server.js` 可启动；POST /api/pages（带 token）→ GET /s/<id>/ 门禁页 → unlock 正确码拿内容、错码 403（node --test 内起真实服务断言）。

### S-02 门禁与限速完善
- 里程碑 M1 ｜ 依赖 S-01 ｜ 必读：docs/04 §6.1（unlock 契约与 cookie 属性逐字）、docs/03 §7
- **目标**：unlock 全契约：限速（UNLOCK_RATE_LIMIT 次/分/IP，内存滑窗即可）、429 响应、会话 cookie（Path=/s/<id>、HttpOnly、SameSite=Strict、Max-Age=86400）、码比对恒定时间。
- **验收**：test：第 6 次错误尝试 429；正确码后携 cookie 直接 GET 拿内容；cookie 属性逐项断言；不同 id 的 cookie 互不放行（Path 隔离断言，HTML 直传安全边界的落点）。

### S-03 REST 契约补齐
- 里程碑 M1 ｜ 依赖 S-02 ｜ 必读：docs/04 §1（错误码总表）/§6.1 全部端点
- **目标**：补齐 PUT（更新+版本递增+可改码）、DELETE（软删 7 天）、GET meta；错误响应统一 `{error,message}`；413 体积上限（MAX_PAGE_MB）；RETAIN_VERSIONS 滚动清理。
- **验收**：契约测试逐端点：每个错误码至少一条用例（400/401/404/409/413/429）；PUT 后 version=2 且 v1.html 仍在；DELETE 后 GET /s/ 404、7 天恢复逻辑用时间注入测试；软删数据未物理删除（数据丢失红线断言）。

### S-04 Docker 部署脚本
- 里程碑 M2 ｜ 依赖 S-03 ｜ 必读：docs/03 §4、mdshare server/deploy.sh
- **目标**：Dockerfile（多阶段、非 root 运行）+ `deploy.sh <user@host> <port> <token>`：build→传输→run，数据卷挂 data/，重复执行=升级且数据不动。
- **验收**：本地 `docker build` 成功、容器内 curl 冒烟通过；deploy.sh shellcheck 通过；**真机部署验收需人工提供目标服务器**（部署前后按全局规则读写 `~/.claude/infrastructure.md`——写入卡内提醒执行 agent）。

### S-05 服务端回归套件
- 里程碑 M1 ｜ 依赖 S-03、P-01 ｜ 必读：docs/04 §6.1、docs/06 P-01
- **目标**：`server/test/` 端到端回归：CLI(P-01 selfhost 适配器) ↔ server 真联调脚本化（起真实 server → publish → 更新 → unlock → unpublish 全链路）。
- **验收**：`npm test` 一条命令跑全链路且可重复执行（每次随机端口+临时 DATA_DIR）；该套件进 CI（G-02 引用）。
