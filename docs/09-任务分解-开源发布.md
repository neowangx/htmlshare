### G-01 README 与文档
- 依赖 K-05 ｜ 必读：docs/01（定位与 F 清单）、docs/02 §3（截图对应线框）、docs/00 D4（诚实声明）
- **目标**：面向两类读者的 README（中英双语，英文在前）：30 秒理解（一行安装→一句话用→链接效果图）+ 深入者（模板/风格表、目标对比表 vercel/cf/selfhost/cloud、访问码强度诚实说明、自托管指南链接 server/README）。
- **步骤**：效果图用 docs/05 样例 A/B 真实产出截图（增强/忠实对比、门禁页、4 风格拼图）；FAQ 收录 docs/02 E1~E8 的用户侧问法。
- **验收**：README 内每条命令逐条真实执行通过（脚本 doctest 式抽取验证）；死链检查；效果图存在于 examples/screenshots/。

### G-02 GitHub 仓库与 CI
- 依赖 C-01（CI 部分建议 M1 结束即启用）；Q4 最晚确认点 ｜ 必读：docs/00 Q4、docs/03 §5
- **目标**：公开仓库（Q4 推荐：个人账号 `htmlshare`）+ Actions CI：push/PR 跑 `npm test`（含 server 回归 S-05），Node 20/22 矩阵。
- **步骤**：`gh repo create`（无 gh 登录则 BLOCKED→NEEDS_HUMAN）；推送；badge 入 README；分支保护 main（CI 必绿）。
- **验收**：CI 在 GitHub 上真实绿（以 `gh run list` 断言）；私有 cloud 仓不在公开远端（remote 检查断言，防泄漏红线）。

### G-03 examples 与一键体验
- 依赖 G-01 ｜ 必读：docs/05 §7（样例即素材）
- **目标**：`examples/`：样例 A/B/C 的 md 源 + 各自产出 HTML（4 风格各一份代表作）+ `examples/README.md` 索引；新用户不装任何东西也能在浏览器打开体验产物。
- **验收**：产出 HTML 全部单文件零外链（复用 C-04 体检脚本批量断言）；examples/README 中每个链接可达（相对路径检查）。
