# 01 产品需求文档（PRD）

> 必读前置：docs/00-决策记录.md（本文引用 D-xx/Q-xx 不再重复论证）

## 1. 一句话定位

**htmlshare：让 AI 的产出物一句话变成精心设计的网页并拿到可发送的分享链接。** 开源、跨 agent、差异化主打「网上发布共享」（D1）。

## 2. 目标用户

与 AI 高频交互、经常需要向他人共享信息的人：

| 画像 | 典型场景 | 关键诉求 |
|------|---------|---------|
| 用 AI 干活的产品/运营/顾问 | 会议纪要、方案、报告发给同事客户 | 体面、易读、微信一键发送、别让外人乱看 |
| 用 coding agent 的开发者 | 原型 HTML、技术方案、评审文档给人过目 | 零配置、命令行一步到位、链接稳定 |
| 内容创作者/研究者 | 长文、调研、教程分享 | 排版好看、读者抓得住重点 |

共同特征：**产出在 agent 对话里，读者在微信/Slack/邮件里**——中间缺一座「一键、体面、可控」的桥。

## 3. 核心场景（用户来完成的三件事)

1. **发文档**：「把这份纪要整理好发给客户」→ agent 转成为阅读优化的交互网页 → 返回链接 + 访问码，粘贴即发。
2. **发原型**：「把这个 demo 页发出去」→ HTML 原样托管（字节级保留）→ 链接 + 访问码。
3. **更新已发的**：改了源文件再说一声 → 链接不变、内容更新、旧版留底（D8）。

## 4. 功能清单

### P0（M0~M1，MVP：达到并超越 mdshare 现有能力）
- F1 md → 忠实 HTML：与原文一致、单文件自包含（D12），sanitize 白名单净化。
- F2 md → 增强 HTML：宿主 agent 按模板/风格再表达（D6），TL;DR、重点面板、折叠/卡片等交互组件；页面内置「原文 ⇄ 增强」开关。
- F3 HTML 直传：`.html` 输入原样托管，不做任何转换。
- F4 自托管发布：上传到用户自己的 htmlshare 服务端，4 位码门禁 + 限速（D4），稳定链接 + 版本留底（D8）。
- F5 Claude Code skill：`SKILL.md` 一句话触发，输出「链接 + 访问码」话术可直接转发。
- F6 模板/风格骨架：通用文档模板 + 2 风格，为 P1 扩展打底（D5）。
- F7 零参数发布：`htmlshare publish <文件>` 无任何必填参数。

### P1（M2~M4，差异化：多平台 + 多 agent + 开源发布）
- F8 Vercel 发布：用用户自己的免费额度，客户端加密门禁（D4）。
- F9 Cloudflare Pages 发布：同上。
- F10 目标自动探测与首配引导（D7）。
- F11 静态页客户端加密（AES-256-GCM，8 位访问码）与 `--public`。
- F12 管理命令：`list`（已发清单）、`unpublish`（撤回）。
- F13 完整模板库（5）× 风格库（4），AI 自选 + 手动指定（D5）。
- F14 一行安装到 Claude Code / Codex / OpenClaw / Hermes（D11）。
- F15 高级设置：config 设默认模板/风格/目标/密码策略。
- F16 GitHub 开源发布：README、示例、CI、v0.1.0。

### P2（M5，商业化）
- F17 云托管服务：CLI 设备码注册登录，免 VPS 发布（D9/D10）。
- F18 免费额度 + Pro 订阅（Q3）、配额执行。
- F19 用户控制台：查看/删除/重置访问码。
- F20 计费接入（Q2）。
- F21 基础访问计数（仅云端，仅计数不做分析）。

## 5. 明确不做（YAGNI 清单）

- ❌ 在线编辑器 / WYSIWYG——编辑发生在 agent 里，不在网页上。
- ❌ 评论、协作、多人团队空间。
- ❌ 社交发现、广场、推荐流——这是分享工具，不是内容平台。
- ❌ docx/pdf/pptx 等输入格式（只收 md 和 html）。
- ❌ 移动端/桌面客户端。
- ❌ 博客引擎 / SEO 站点生成器（默认对分享页 `noindex`）。
- ❌ 自定义 JS 插件系统。
- ❌ 超出计数的访问分析（热图、来源分析等）。
- ❌ 自建 CDN / 对象存储。

**红线不减**（即便追求极简）：输入净化（sanitize 白名单）、上传鉴权、数据丢失防护（版本留底、删除二次确认）、无障碍（语义化标签、对比度、键盘可达）。

## 6. 非功能指标

| 指标 | 目标 |
|------|------|
| 首次使用摩擦 | 安装 → 拿到第一个分享链接 ≤ 3 分钟；必填配置项 = 0（有 Vercel/自托管任一凭据时） |
| 转换耗时 | CLI 确定性部分 ≤ 3s（不含 agent 增强时间） |
| 页面性能 | 常规文档产物 ≤ 2MB；单文件零外链，首屏 ≤ 1s |
| 可靠性 | 增强失败/agent 缺席 → 忠实版照常发布（D6）；云服务不可用 → 其他目标不受影响（D9） |
| 安全 | 服务端目标限速 ≤ 5 次/分/IP；静态目标加密强度按 D4；HTML 直传页与门禁隔离声明 |
| 无障碍 | 正文对比度 ≥ 4.5:1；全键盘可操作；语义化标题层级 |

## 7. 成功指标

- **北极星**：周发布次数（开源遥测不做，以云端数据 + GitHub 反馈估计）。
- 安装转化：README 访问 → 安装 → 首次发布漏斗（以 issue/讨论质性判断，MVP 不埋点）。
- GitHub stars / forks 增长（分发健康度代理指标）。
- 云服务：注册数、免费→Pro 转化率、月流水（M5 后）。

## 8. 与 mdshare 的关系

htmlshare 是独立产品（D1/D2）：复制其代码基线，配置目录、manifest、服务端全部独立。用户可同时安装两者互不干扰。mdshare 不再演进新功能（用户自行决定何时退役）。
