# 02 用户体验设计

> 必读前置：docs/00（引用 D-xx）、docs/01（引用 F-xx）
> htmlshare 的"界面"有两处：**CLI/agent 对话**（发布侧）与**产出网页**（阅读侧）。两侧分别设计。

## 1. 设计准则

1. **为读者优化，不为作者炫技**：增强版的唯一目的是让收到链接的人更快抓住重点、更好理解——不是展示花哨组件。组件（折叠/标签页/卡片）只在确实降低认知负担时使用（规则见 docs/05 §4）。
2. **零学习成本**：发布侧一句话触发、零必填参数（F7）；阅读侧打开即读，门禁页只有一个输入框。
3. **诚实的边界**：访问码是轻量门禁不是保险柜——强度声明写进门禁页脚注与 README（D4）。
4. **永远有降级**：增强失败给忠实版，云挂了给其他目标，断网给明确的重试指引。任何失败都不能吞掉用户的内容。
5. **克制动效**：只用于状态反馈（展开/切换），无装饰性动画。

## 2. 信息架构

```
发布侧（CLI / agent 对话）
├─ publish <file> [--target --template --style --code --public --title]
├─ list                     已发清单（标题/目标/链接/更新时间）
├─ unpublish <file|id>      撤回（二次确认）
└─ 首次使用引导（无目标凭据时自动出现，只出现一次）

阅读侧（产出网页，全部单文件 D12）
├─ 门禁页（有访问码时的第一屏）
└─ 内容页
   ├─ 页头：标题 · 更新时间 · [原文 ⇄ 增强] 开关（md 输入才有）
   ├─ TL;DR 面板(增强版，≤5 条)
   ├─ 正文（模板决定结构，风格决定视觉）
   └─ 页脚：made with htmlshare（可 config 关闭）· 门禁强度说明
```

## 3. 文字线框稿

### 3.1 CLI：首次发布（无任何配置，检测到 vercel 已登录）

```
$ htmlshare publish 会议纪要.md
✓ 转换完成（忠实版 + 增强版 · 模板:会议纪要 · 风格:临床卡片）
· 未设置默认发布目标，检测到 Vercel 已登录，本次使用 Vercel（已记住，
  可用 htmlshare config target 更改）
✓ 已加密（访问码门禁）
✓ 发布成功

  链接  https://htmlshare-pages-xxxx.vercel.app/s/k3f9m2/
  访问码  7XK4-Q2NM

  更新：改完源文件再执行同一命令，链接不变。
```
状态：成功 / 失败（每一步失败有独立文案，见 §5 异常分支）。

### 3.2 CLI：首次使用且无任何凭据（一次性引导）

```
$ htmlshare publish 方案.md
✓ 转换完成
! 还没有可用的发布目标，三选一：
  1) Vercel      免费，需先执行:  npx vercel login
  2) Cloudflare  免费，需先执行:  npx wrangler login
  3) 自托管      自己的服务器，执行:  htmlshare config selfhost
  完成后重新执行本命令即可。转换结果已暂存，不会重做。
```

### 3.3 门禁页（阅读侧第一屏）

```
┌────────────────────────────────────┐
│            （风格化标题区）           │
│         此页面需要访问码             │
│      ┌──────────────┐  [ 打开 ]     │
│      └──────────────┘               │
│   状态: 空闲 / 校验中 / 错误(码不对,   │
│         请与分享者确认) / 剩余尝试提示 │
│  ‧ 轻量保密说明（一行小字，可展开）    │
└────────────────────────────────────┘
```
- 服务端版：POST 校验，429 时提示「尝试过频，稍后再试」。
- 静态加密版：本地解密，错误即「访问码不正确」；解密中显示进度（KDF 约 0.5~1s）。
- 输入框自动聚焦、支持回车提交、粘贴自动去除空格连字符。

### 3.4 内容页（增强版，以「会议纪要」模板 × 临床卡片风格为例）

```
┌──────────────────────────────────────────┐
│ 产品评审会纪要        2026-07-04  [原文⇄增强] │
├──────────────────────────────────────────┤
│ ▍TL;DR                                    │
│ • 定了：Q3 主打 X 功能                      │
│ • 待定：预算需 CFO 批（本周五前）            │
│ • 你要做：见「行动项」中 @你 的两条          │
├──────────────────────────────────────────┤
│ [结论卡] [行动项卡:负责人+期限] [分歧与遗留]  │
│ [讨论过程 ▸ 折叠，点开看完整脉络]            │
├──────────────────────────────────────────┤
│ made with htmlshare · 轻量访问码保护        │
└──────────────────────────────────────────┘
```
交互状态：开关切换瞬时（两版都已内联）；折叠区记忆展开状态（sessionStorage）；打印样式自动展开全部折叠。

### 3.5 CLI：更新与撤回

```
$ htmlshare publish 会议纪要.md          # 同一文件 → 复用 id
✓ 已更新（链接与访问码不变，旧版留底）
  https://.../s/k3f9m2/

$ htmlshare unpublish 会议纪要.md
! 将撤回该分享（链接立即失效，云/自托管保留 7 天可恢复，静态目标立即删除）
  确认？ [y/N]
```

## 4. 完整用户场景模拟

**人物**：李哲，32 岁，B 端产品经理，Claude Code 重度用户，不懂前端部署。
**时刻**：周三 16:40，刚开完客户需求评审会，17:30 前要把纪要发到客户微信群。

1. 16:41 对 Claude Code 说："把刚才的会议记录整理成纪要，用 htmlshare 发出去给客户看。"
2. agent 整理 md → 触发 skill → 判定内容类型=会议纪要，选模板「会议纪要」、风格「临床卡片」（客户场合，克制专业）→ 生成增强片段 → CLI 组装、加密、发布到已记住的 Vercel 目标。
3. 16:43 agent 回复：链接 + 访问码 + 一句可直接转发的话术。李哲粘到微信群，另发访问码。
4. 16:50 客户打开：门禁页输码 → TL;DR 三条先入眼 → 行动项卡各自认领。客户老板只扫了 TL;DR，10 秒抓到结论。
5. 17:10 客户指出一处金额写错。李哲让 agent 改源文件重发同一命令——**链接不变**，群里无需重发。

**验证结论**：全程 0 配置操作、0 次离开 agent 对话、读者 0 学习成本。此场景走查确认了 D6（agent 增强）、D7（记住目标）、D8（稳定链接）三个决策成立。

## 5. 异常分支（≥5，全部有明确出口）

| # | 异常 | 系统行为 | 用户侧表达 |
|---|------|---------|-----------|
| E1 | agent 增强产出校验失败（Schema 不符/危险标签） | 重试 1 次仍失败 → 仅发布忠实版 | 「增强版生成未通过校验，已发布原文版（同样干净可读）」 |
| E2 | 断网 / 上传超时 | 转换产物暂存本地，指数退避重试 2 次后停止 | 「网络不可用。产物已暂存，恢复后重跑同一命令即可（不会重新转换）」 |
| E3 | Vercel 未登录 / token 过期 | 探测阶段发现，不消耗转换 | 「Vercel 凭据失效，请 npx vercel login；或 --target 换用其他目标」 |
| E4 | 发出去才发现内容含敏感信息 | `unpublish` 立即失效链接（静态目标重部署删除，秒~分钟级生效） | 撤回确认提示中写明各目标生效时长与「静态平台可能有短暂 CDN 缓存」的诚实告知 |
| E5 | 忘记访问码 | `list` 命令显示本地 manifest 中记录的码；服务端目标另可控制台重置 | 「访问码：XXXX（本地有记录，不用记）」 |
| E6 | 单页超限（图片太大 >5MB） | 发布前体检告警，可 `--force` 继续（云端仍受配额硬限） | 「页面 8.2MB，微信内打开会很慢。建议压缩图片；仍要发布加 --force」 |
| E7 | 云服务宕机 / 停止运营 | 适配器超时快速失败，提示切换目标；其余目标零影响（D9） | 「云服务暂不可用，可 --target vercel 立即改发；已发内容可随时迁至自托管」 |
| E8 | 宿主 agent 无增强能力（headless/弱模型） | CLI 侧检测无增强片段输入 → 直接走忠实版 | 输出正常链接，注明「本次为原文版」 |

## 6. 无障碍与阅读细则（红线）

- 全部模板：语义化 `h1~h3`/`nav`/`main`；折叠用 `<details>` 原生元素保证无 JS 也可读。
- 对比度 ≥ 4.5:1（四种风格逐一校验，写入 C-06/C-14 验收）。
- 门禁页与开关全键盘可达；`prefers-reduced-motion` 时禁用过渡。
- 深浅色自适应（`prefers-color-scheme`），暗色科技风格除外（固定深色，注明）。
