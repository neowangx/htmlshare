### V-01 私有仓初始化与多租户模型
- 依赖：S-03 完成（协议基线稳定）｜ 必读：docs/04 §6/§7.2、docs/03 §5 末段、docs/00 D9/D13
- **目标**：`htmlshare-cloud/` 私有仓：复制开源 server/ 起步，加 SQLite（better-sqlite3）索引层（users/pages/versions/usage_events per docs/04 §7.2），页面内容仍存文件系统。
- **步骤**：git init（私有，不推公有远端）；db.js 建表+迁移函数（幂等）；pages 写入同时落 SQLite 与文件；开源仓的 server 契约测试**原样拷入必须全绿**（同构性回归底座）。
- **验收**：拷入的开源契约测试全绿（一行不改）；建表幂等（跑两次）；SQLite 记录与文件系统一致性断言。

### V-02 设备码登录
- 依赖 V-01 ｜ 必读：docs/04 §6.2（逐端点）、docs/02 §3（体验基调）
- **目标**：/api/auth/device、/api/auth/token、/activate 激活页（服务端渲染极简表单：userCode+邮箱→邮件验证码→绑定）。
- **步骤**：设备码 10 分钟过期、轮询 428/410 分支；邮件发送走 SMTP_URL（本地测试用文件伪投递驱动，SMTP 空时写 ./outbox/ 便于测试与人工验收）；token 为随机 32B，库存 hash；上传端点接入用户级 Bearer 鉴权（原全局 token 保留为 admin）。
- **验收**：test 全流程（含三分支）无真实 SMTP 跑通（outbox 驱动断言邮件内容含验证码）；token 泄漏面断言（响应/日志无明文 hash 前 token 只出现一次）；未验证邮箱不得下发 token。

### V-03 配额与用量
- 依赖 V-02 ｜ 必读：docs/04 §6.2（/api/me）、§7.2（usage_events）
- **目标**：发布/查看计数入 usage_events；/api/me 返回 plan 与 usage；页数与单页体积按 plan 限制读 env（FREE_*/PRO_*）。
- **验收**：达到 FREE_PAGES 后 POST 返回 403 QUOTA_EXCEEDED；删页后配额释放；/api/me 数字与实际一致（test 造数断言）；view 计数不阻塞响应（异步写，失败仅日志）。

### V-04 计划限制与回收策略
- 依赖 V-03 ｜ 必读：docs/00 Q3（推荐值）、docs/04 §1（402/403 语义区分）
- **目标**：plan 字段驱动全部限制；不活跃回收（180 天未访问的免费层页面：先邮件预警 14 天再软删）；管理脚本 `node tools/set-plan.js <email> pro`（V-07 前的人工开通通道）。
- **验收**：限制矩阵测试（free/pro × 页数/体积）；回收流程时间注入测试（预警邮件入 outbox → 期满软删 → 期内访问则豁免）；set-plan 脚本幂等。

### V-05 用户控制台（最小）
- 依赖 V-02 ｜ 必读：docs/01 F19（范围就这三件事：查看/删除/重置码）、docs/02 §1（准则）
- **目标**：`/console` 服务端渲染页面（登录=邮箱验证码，复用 V-02 邮件通道）：列自己的页（标题/链接/更新时间/用量条）、删除（二次确认）、重置访问码。
- **约束**：零前端框架、单 CSS 内联、无 JS 也可完成全部操作（表单提交）——红线：无障碍与极简。
- **验收**：test：未登录 302；他人页面不可见/不可删（越权断言）；重置码后旧码 unlock 403 新码 200；页面 HTML 过 W3C 基本结构断言（有 title/main/form label）。

### V-06 部署上线
- 依赖 V-04、V-05；Q1 最晚确认点 ｜ 必读：docs/03 §4/§6、`~/.claude/infrastructure.md`（**部署前必读，部署后必更新——全局铁律**）
- **目标**：Docker + Caddy（自动 TLS）部署到自有服务器，域名用 Q1 确认值（未确认则 BLOCKED 记 NEEDS_HUMAN，先在临时端口完成除 TLS 外全部验收）。
- **验收**：生产 URL 全链路冒烟（设备码注册→publish→门禁→console 删除）；HTTPS 强制跳转；重启容器数据不丢；**弱耦合演练：停掉 cloud 容器，开源 CLI 对 vercel/selfhost 目标发布不受任何影响（脚本化断言，D9 验收的最终落点）**。

### V-07 计费接入
- 依赖 V-06；**截止点守卫：Q2 未确认不得开工（渠道选择过点难改）**｜ 必读：docs/00 Q2/Q3、docs/10 成本测算
- **目标**：按 Q2 确认的渠道接订阅（Stripe 则 Checkout+webhook 改 plan；国内过渡方案则收款码+人工 set-plan 的产品化页面）。
