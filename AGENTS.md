# AGENTS.md — htmlshare 执行启动文件

> 你是执行开发的 Coding Agent。读完本文件即可开工，全程无需询问规划者。
> 规划文档在 `docs/`，契约以 `docs/04` 为准，决策以 `docs/00` 为准。

## 使命

把 htmlshare 从规划变成可用产品：**开源的跨 agent「AI 产出物发布分享」skill**（md/HTML → 精心设计的网页 → 一键分享链接 + 访问码），外加弱耦合的增值云托管。

**完工的定义**：TASKBOARD.md 全部任务卡 done（blocked 卡按卡壳协议处理完毕）+ 全量测试通过 + M0~M5 每个里程碑出口标准（PLAN.md 末尾）全绿 + 写出 REPORT.md。

## 状态自检（幂等启动）

每次启动（含中断恢复、换模型接手）：
1. 读 `TASKBOARD.md` → 确定每张卡状态，**绝不重做已 done 的卡**。
2. 读 `BLOCKERS.md` 与 `NEEDS_HUMAN.md` → 若有阻塞项已被人类解决（如密钥已提供），把对应卡从 blocked 改回 todo。
3. 从「依赖全部 done 且里程碑最靠前」的 todo 卡继续。

## 开放问题处理协议

开放问题 Q1~Q5 见 `docs/00`，每个都有推荐方案。
1. **交互式会话**：开机时一次性列出尚未确认的 Q（推荐方案+最晚确认点）请用户确认，只问这一次；用户不答就按推荐执行。
2. **无人值守**：直接按推荐方案执行，TASKBOARD 完成说明里标注「按 Q-xx 推荐执行」。
3. **截止点守卫**：取到某未确认 Q 的最晚确认点卡（Q1→V-06、Q2→V-07、Q3→V-04、Q4→G-02）时：只冻结该卡（标 blocked + 记 NEEDS_HUMAN.md），换下一张卡继续。**绝不把未确认决策执行过不可逆点**（尤其 V-07 支付渠道）。
4. 用户确认的唯一形式 = `docs/00` 被修改。执行前重读该文件。

## 原型门禁协议（D15）

U-02 原型包完成后：把启动命令与逐页走查清单（docs/02A §5）写入 NEEDS_HUMAN 并明确通知用户走查。**这是唯一允许停等人类的合法门禁点**——等待期间只做不受门禁约束的卡（C-01~C-04、C-07~C-09、P/S/K 系列），绝不提前开工 C-05/C-06/C-10~C-14。收到反馈 → 修原型 → 再交付；用户确认记入 docs/00 变更记录后 U-03 才标 done。界面实现卡的验收必须逐条跑 docs/02A §7 清单，产出以原型为黄金标杆。

## 首次启动步骤（仅当仓库还是纯文档状态）

1. 本目录即开源仓工作目录。按 `docs/03 §5` 建目录结构（C-01 卡就是这件事，从它开始）。
2. `git init`（若未初始化）；首个 commit 提交规划文档基线。
3. `npm install`，跑通空测试（`npm test`）验证环境。
4. 参考代码基线在 `~/Documents/claudeMini/mdshare/`——**只读**。红线：严禁修改 mdshare 目录任何文件（docs/00 D1）。
5. V 系列任务在兄弟目录 `../htmlshare-cloud/`（V-01 负责初始化，私有）。

## 工作主循环协议

```
while TASKBOARD 存在非 done 且非 blocked 的卡:
    取卡: 依赖全部 done 且里程碑顺序最靠前的卡; 标记 doing
    读该卡标注的必读文档 → 实现 → 真实执行该卡的验收标准
    验收不过 → 修复重试(同卡最多3轮)
    验收通过 → TASKBOARD 标 done + 一行完成说明 + git commit(消息带卡号)
    里程碑内全部 done → 执行该里程碑出口标准回归(PLAN.md 末尾) → 在 TASKBOARD 记录里程碑达成
全部完成 → 跑全量测试 + docs/10 §5 上线前检查清单 → 写完工报告 REPORT.md
```

## 子 Agent 并行规则

- 可并行组（依赖图见 docs/06~09）：{C-02,C-03}、{C-10,C-11,C-12,C-13}、{P-02,P-03}、{K-03,K-04}、{S-04,S-05}、{V-05 与 V-03/V-04}。
- 每个子 Agent 的输入 = 任务卡全文 + `docs/04` 路径 + 本文件的「诚实汇报铁律」「契约纪律」两节。
- 并行修改代码用 git worktree（或分支）隔离；合并前必须重跑双方卡的验收。
- 不支持子 Agent 的执行环境：按依赖图串行，协议不变。

## 卡壳协议（防死循环）

- 同一卡修复 3 轮验收仍不过 → 状态改 blocked，原因写入 `BLOCKERS.md`（卡号/现象/已试方案），换下一张卡。
- 需要人类才能解决的事（登录、密钥、域名、付费、法务文本、真机视觉走查）→ 追加到 `NEEDS_HUMAN.md`（注明阻塞哪些卡），**继续其他不受阻塞的工作，绝不空转等待**。
- 全部剩余卡都 blocked 时才停机，停机前把 BLOCKERS.md 整理成给人类的行动清单。

## 诚实汇报铁律

- 验收命令必须真实执行，输出证据留在 commit message 或测试文件中；**禁止未验证即标 done**。
- 测试失败如实记录；不许为过验收删除、跳过（skip）或弱化测试断言。
- 「需人工」的验收项：完成可自证部分后标 done 并在完成说明注明「人工项待验：xxx」，同时登记 NEEDS_HUMAN.md。

## 契约纪律

- `docs/04` 是唯一契约。实现中发现契约有误/需变更：**先改 docs/04 + 在 docs/00 变更记录留痕（写明动因），再改代码**。禁止代码悄悄偏离文档。
- 枚举值（模板/风格/槽位/错误码）以 docs/04 §1/§8 为唯一拼写，任何文档与代码不得出现别名。
- 全局规则（来自用户环境）：任何真实部署前读 `~/.claude/infrastructure.md` 取凭据与现状，部署成功后更新它。
